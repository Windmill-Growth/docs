## Before you connect anything

Owners and admins set up integrations and change delivery settings.

## Available integrations

The current public integrations are:

- **Slack**
- **Webhook**

## Slack

Use Slack when you want lead alerts and workspace activity notifications inside a channel.

To set it up:

1. Open **Integrations**.
2. Connect your Slack workspace.
3. Select the channel that should receive notifications.
4. If you want a private channel, invite `traxy.ai` into that channel first with `/invite @traxy.ai`.
5. Enable the **Qualified Lead Detected** trigger once the channel is set.
6. Decide whether you want auto-send on or off.
7. If you turn on auto-send, set the ICP match threshold that should trigger delivery.
8. Send a test event before you rely on it.

If you connect Slack but do not select a channel, Slack stays disabled.

## Webhook

Use Webhook when you want `traxy` to send qualified lead events to another system.

To set it up:

1. Open **Integrations**.
2. Add an integration name.
3. Add a secure `https://` endpoint.
4. Add an auth header if your receiver expects one.
5. Enable the **Qualified Lead Detected** trigger.
6. Decide whether you want auto-send on or off.
7. If you turn on auto-send, set the ICP match threshold that should trigger delivery.
8. Open **Payload & Help** if you want to review the sample payload and signing guidance.
9. Send a test payload before you rely on it.

Webhook delivery is designed for qualified lead workflows. Keep the receiver focused on lead routing and downstream automation.

Each workspace supports one active Webhook integration. If you need to change destinations, update the existing Webhook instead of creating parallel ones.

The Webhook UI includes a payload browser and sample JSON. The payload includes event metadata plus lead, profile, company, contact, and engagement data.

Webhook requests are signed. Each request includes `X-traxy-Signature` and `X-traxy-Timestamp` so your receiver can verify that it came from `traxy`.

## Manual vs automatic delivery

You can work in two ways:

- **Manual delivery** from the leads table when you want to review each lead first
- **Automatic delivery** when you want `traxy` to route qualified leads as soon as they meet your threshold

Automatic delivery is non-retroactive. It sends leads that qualify after auto-send is turned on.

Use test sends before you go live. Test sends and manual sends are single-attempt checks. Automatic delivery retries after temporary failures and may pause if repeated failures are not fixed.

If you send manually from **Leads**, you may see states such as queued, delivered, or failed. Fix the underlying integration issue first, then retry from the lead.

Review automatic settings carefully before you enable them for a shared workspace.
